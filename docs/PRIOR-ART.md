# Prior art — adopted principles and their provenance

Underpainting's generation approach adopts **principles** that are observable, at the category level, in production AI design tools — for example, Anthropic's public launch materials for Claude Design describe design-system grounding, inline-commented output, generated controls, and handoff to Claude Code. This file records each adopted principle in our own words, why it fits *our* constraints, and what enforces it here.

**Provenance rules (ADR-008, applied with full force):** only public materials are cited; principles may be learned, text may never be taken. No extracted prompt text — quoted or paraphrased — lands anywhere in this repository: docs, ADRs, issues, comments, or commit messages. Our prompts are written from BUILD_BRIEF.md §7 and the Validator's checks, and quality is pulled up by the eval harness, not by borrowed wording.

| Principle (our words) | Why it fits our constraints | Enforcement here |
|---|---|---|
| **A1 — Token-block-first inline styling.** One `<style>` block (tokens, `@font-face`, `@keyframes`, reset) at the top; every element styled inline via `var(--token)`. | Streaming: the token block arrives first, so every later element paints correctly on arrival. Grounding: the block is copied from `.design/system/tokens.css`, so BYOK output uses the repo's real brand values. | Validator rule A1 (M1 item 6); golden check `a1-token-styling`; ADR-007 |
| **A2 — Editability structure.** Every visible text run is its own leaf element; repeated structure is written out literally; scripts never construct layout or content. | Free local edits (P4): a canvas click must resolve to exactly one source span so edits never round-trip through the model — the user's money is not spent re-generating what they typed. | Validator rule A2; golden check `a2-literal-repetition`; the "I can't edit that part" failure mode is a golden-eval case |
| **A3 — Self-containment.** No CDN scripts, external fonts, or remote images. | The artifact sandbox stays airtight (P6: no network inside the iframe) and artifacts render identically offline — local-first means no third-party fetch at view time. | Validator rejects any external reference; renderer strips `<link>`; CSP blocks at runtime; golden check `a3-self-contained` |
| **A4 — Streaming-safe form.** Nothing may depend on content that hasn't streamed yet; late-arriving elements carry explicit dimensions. | The canvas paints from the first token (≤3s p95 to first paint); layout must not jump while the tail streams. | Validator rule A4; morph renderer tested for mid-stream sanity (`test/unit/artifactDom.test.ts`) |
| **A5 — Scaffolds are copied, never regenerated.** Boilerplate shells live as versioned files and are copied in; the model generates content into them. | BYOK cost pressure: regenerating solved boilerplate spends the user's money on nothing, and copied scaffolds can't drift or hallucinate. | `/scaffolds` + copy-in mechanism and validator awareness (M1 item 7) |
| **A6 — Content quality.** No filler, no invented statistics or decorative data, no unrequested material; accessible by default. Requested-but-unspecified content (e.g. "a short feature list") is in scope and not flagged. | Cost transparency loses meaning if tokens go to padding; trust in output requires that nothing poses as real data. The unrequested/unspecified boundary keeps the correction loop from burning retries on compliant output. | Deterministic-subset checks in evals (scoring rule in `evals/README.md`); prompt rule 9 in `prompts/core.md` |
| **A7 — Surgical edits.** A targeted change touches only what was asked; untouched content survives byte-identical where possible; broader improvements are suggested, never applied. | Refinement is the dominant loop; unrequested churn destroys the user's prior review *and* costs money to re-verify. Diff minimality is measurable, so it's a merge bar, not a vibe. | Validator rule A7 (M1 item 6); golden diff-minimality case (M1 item 10) |
| **Clarify before spending.** When a request is ambiguous about material that would cost tokens to produce, ask (or take the minimal interpretation) rather than padding. | It is the user's money (P3); an unwanted elaborate generation is worse than a one-line question — and the free/paid boundary must stay legible (P4). | Prompt rule 9 in `prompts/core.md`; A6 eval checks; chat-panel UX (M1 item 3) |
| **Cheap-model verifier pass.** Validation/correction runs on a fast, cheap model with minimal context, separate from the generator, bounded by the retry cap. | Deterministic checks catch structure; a small verifier catches the rest at a fraction of generation cost — and the hard cap keeps a failing loop from becoming a bill (§9). | Per-task validation-model setting (M1 item 1); correction loop wiring (M1 item 6); `CORRECTION_RETRY_CAP` |
